What is a state?

Think for a second...


  • It's summer time
  • You are sitting by the lake
  • Under a large beach umbrella
  • And your husband is grilling some bacon

Your state is: pretty relaxed.

Think for another second...


  • You are in a meetup
  • Talking in front of 50+ people
  • Trying to explain what is state
  • Everyone is staring at you

Your state is: pretty nervous.

This means state of your mind changes based on the situations

So what is a state?

State is something

  • that might change
  • like: your mood, your confidence level
  • like state of water: ice, water, stream
  • or emotion about your partner

What is changing in this pic?

In an application

A lot of information stored in the memory

  1. Variables
  2. Classes
  3. Data Structures
  4. Data from server

All the stored information, at a given instant in time, is called the application state.

In a React App

You can think of two level of states

  1. Application State: Global State
    • Like: Session storage/ store in Redux
    • Think it like a database
    • Any component, anywhere in the app can access
  2. Component State
    • Within that specific component
    • Can only be updated within that component
    • Can be passed down to its children via props

Stateless

I've seen and talked to developers lately who are obsessed with statelessness to the point of distraction. Its nice to have statelessness for re-usability but its not realistic to pursue that goal in every situation over every other goal unless you have a ton of resources to do that for some reason, like scaling.

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );
